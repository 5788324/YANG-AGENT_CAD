# 工作计划

## 总目标

开发一个 AutoCAD AI Agent，让代码小白和 CAD 新手可以通过自然语言完成 AutoCAD 当前图操作、批量处理、插件沉淀、日志追踪和错误回退。

## 阶段 0：项目初始化，预计 0.5-1 天

- 初始化 GitHub 仓库结构。
- 建立 Codex / Antigravity 通用工作规范。
- 建立每日工作日志和交接日志机制。
- 建立错误码、任务账本、回滚规范。
- 建立插件箱 manifest 规范。

验收标准：

- 新 AI 打开仓库后能知道下一步做什么。
- 每次工作能开始同步、结束提交。
- 用户不需要看代码也能理解项目状态。

## 阶段 1：安全可回滚 MVP，预计 5-7 个工作日

- 创建 Python 项目骨架。
- 实现 CLI 入口 `yang-cad-agent doctor`。
- 实现 AutoCAD 环境检测：AutoCAD、accoreconsole、Python、依赖、工作目录。
- 实现 task ledger：每次任务生成 task_id 和 JSON 日志。
- 实现备份和回滚模块。已完成基础版。
- 实现 LISP 静态校验器。已完成基础版。
- 实现当前图 LISP 投喂原型。
- 实现 accoreconsole 批量执行原型。已完成 dry-run scaffold。
- 准备第一个端到端样例：查询/清理/批量 dry-run。已完成 CLI dry-run 冒烟测试。

验收标准：

- 任何修改任务都有任务日志。
- 批量修改前自动备份。
- 失败时返回错误码和日志路径。
- 可按 task_id 回滚。

## 阶段 2：插件箱 Beta，预计 1-2 周

- 实现插件箱目录结构。已完成基础目录。
- 实现插件 manifest 校验。已完成基础版。
- 实现插件安装、列出、执行、收藏。
- 支持 AI 临时 LISP 转正式插件。
- 支持插件风险分级和执行确认。
- 把常用 CAD 操作做成首批插件。

验收标准：

- 用户可以让 AI 把有用脚本加入插件箱。
- AI 可以优先调用已有插件。
- 插件支持当前图和批量任务两类执行。

## 阶段 3：MCP 和多客户端适配，预计 1 周

- 实现 MCP Server 基础工具。
- 适配 Codex 项目规则。
- 适配 Antigravity 工作方式。
- 输出统一 runtime contract。
- 封装受控工具，避免泛用 shell。

验收标准：

- Codex / Antigravity 都能按同一套规则工作。
- MCP 能列出插件、执行插件、查询任务、回滚任务。

## 阶段 4：稳定化 V1，预计 2-3 周

- 增加更多错误模式分析。
- 增加 GitHub Actions 的非 AutoCAD 单元测试。
- 增加本地 AutoCAD 半自动测试清单。
- 增加备份清理策略。
- 增加图纸锁和任务并发控制。
- 完善用户文档和故障排查。

验收标准：

- 日常任务可稳定使用。
- 失败可定位、可重试、可回滚。
- 新 AI 接手不丢上下文。

## 总预估

- MVP：5-7 个工作日。
- Beta：2-3 周。
- V1：4-6 周。
